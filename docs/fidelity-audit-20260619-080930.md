# VibeVoice-MLX vs VibeVoice (PyTorch) — Fidelity Audit

_Generated: 2026-06-19_

Comparison of `vibevoice-mlx` (this repo) against the reference PyTorch implementation
at `../VibeVoice`. The MLX port is inference-only; this audit checks whether it
faithfully reproduces the reference logic, and flags real divergences and genuine
improvements.

The port is a faithful, well-engineered reimplementation. Architecture, conv weight
transposes, causal-cache derivations, CFG math, and the Qwen2.5 backbone all map
correctly. The findings below are the divergences five reviewers found; the two
headline items (#1, #6) were verified directly in source.

## 🔴 Bugs (ranked)

### 1. VAE decoder trim is symmetric, not causal — corrupts final audio (HIGH — verified)

`vibevoice_mlx/model.py:340-343` trims the transposed-conv output symmetrically:
`trim=(K_up-stride)//2; x=x[:,:,trim:-trim]`. The reference is causal
(`trim_right_ratio=1.0`) and trims `K-stride` from the **right only**. The port's own
*streaming* decoder gets this right (`streaming_vae.py:117-121`), so the two MLX
decoders disagree. This symmetric path produces the **final clean audio**
(`generate.py:485`) and the per-token semantic feedback (`generate.py:415`). For odd
ratios (5, 5 in `[8,5,5,4,2,2]`) it also changes output length (~`3200·L+96` vs
`3200·L`), time-shifting every upsample stage.

**Fix:** `pad = K_up - stride; if pad > 0: x = x[:, :, :-pad]` — match the streaming impl.

### 2. DPM-Solver++ penultimate step downgraded to 1st order (MED/HIGH)

`generate.py:150-151` forces 1st-order on the second-to-last step when `num_steps<15`.
In diffusers, `lower_order_second` is a no-op for `solver_order==2` (it still runs
2nd order). The port's own SDE branch (`generate.py:209`) gates only on `is_last`,
confirming the ODE path is inconsistent.

**Fix:** drop `lower_order_second` from the `use_first_order` condition.

### 3. Noise schedule off-by-one + missing `max_beta` clamp (MED)

`generate.py:27` computes `alphas_cumprod[k]=ᾱ(k/1000)`, but the reference
`cumprod(1-betas)` yields `ᾱ((k+1)/1000)` — a one-index shift on every α/σ/λ. The port
also omits the reference's `beta≤0.999` clamp; impact is largest at high noise
(t=999: α≈1.6e-3 vs ref ≈5e-5), shifting the first ODE step.

**Fix:** build `betas → clamp(0.999) → alphas_cumprod=cumprod(1-betas)` exactly as
reference.

### 4. Token constraint only applied for `single_segment` models (MED)

The reference *always* masks output-phase logits to {speech_start, speech_end,
speech_diffusion, eos} (`inference.py:405-419`). The port applies this only when
`config.single_segment` is True (`generate.py:353-361`); the default multi-segment
path leaves all 151,936 vocab logits unconstrained, allowing text tokens to leak into
the audio phase.

**Fix:** build `logit_mask` from those four IDs unconditionally.

### 5. Speaker-ID renumbering merges speakers (MED)

Reference shifts speaker IDs only when global `min(speaker_ids)>0`. The port
decrements every line `max(0,N-1)` unconditionally (`e2e_pipeline.py:255-259`),
collapsing distinct speakers in 0-based scripts.

**Fix:** compute global min over parsed lines; shift only when `min>0`.

### 6. README falsely claims "PyTorch-compatible RNG" (MED — verified)

`README.md:128,142` claims "Pure Python MT19937 + Box-Muller matching `torch.randn()`
exactly." Reality: `numpy_rng.py` is a 4-line `np.random.RandomState` wrapper, **never
imported anywhere**. The live path uses MLX Threefry (`generate.py:126`), which cannot
match torch bit-for-bit.

**Fix:** delete dead `numpy_rng.py` and correct the README, or actually port torch's
CPU `normal_` if parity matters.

### 7. Connector RMSNorm eps mismatch (LOW — cross-flagged twice)

`model.py:396` uses `eps=1e-5`; reference `SpeechConnector` uses `1e-6`
(`modeling_vibevoice.py:61`). Affects every acoustic & semantic embedding fed to the
LLM. Negligible magnitude but a true constant divergence.

**Fix:** `eps=1e-6`.

### 8. Lower-severity divergences

- Negative CFG branch advanced every step vs only on diffusion steps — desyncs neg
  context after segment boundaries in multi-speaker (`generate.py:448-456`). _(LOW)_
- `--no-semantic` silently diverges; reference always adds semantic feedback
  (`generate.py:432`). _(LOW)_
- Encoder drops reference's stride-alignment tail padding → possible ±1 latent frame
  (`vae_encoder.py:28`). _(LOW)_
- `model.load_weights(..., strict=False)` on quantized/per-layer paths masks misnamed
  keys silently (`load_weights.py:318,348`). _(LOW — assertion gap)_
- Default `diffusion_steps=10` vs reference `20`; intentional but out-of-box trajectory
  differs. _(INFO)_
- Whole diffusion head/VAE run in fp16 vs reference bf16 — safe here (activations O(1))
  but a precision note.

## 🟢 Genuine improvements the port made

- **Fused dual-branch LM forward** (`fast_forward.py:69-150`): batches main+neg CFG
  branches into one matmul/layer — real speedup, no correctness cost.
- **Batched CFG** single concatenated diffusion-head forward vs reference's double-batch.
- **fp32 accumulation** for CFG combine, x0 reconstruction, and untied-7B logits —
  guards the bf16→fp16 port against overflow.
- **float64 precomputed α/σ/λ schedule tables** — exceeds reference fp32 precision
  (ironically making bug #3 easy to fix cleanly).
- **Streaming decoder caches `K-stride`** (vs reference `K-1`) — provably sufficient,
  less memory, identical numerics. _And it's the correct trim logic that model.py
  should copy._
- **`mx.compile` functional semantic encoder** + fused T=1 depthwise path — latency
  win, identical math.
- **Selective quantization** correctly scoped to `nn.Linear` in the LLM only;
  embeddings/norms/convs/diffusion-head never quantized.
- **CoreML semantic offload to ANE** for GPU pipelining; KugelAudio auto-detection
  heuristic.

## Recommended priority

Fix **#1** first (it directly degrades every output and is a 1-line change mirroring
the port's own correct streaming code), then **#4** and **#5** (multi-speaker
correctness), then **#2/#3** (diffusion-trajectory fidelity), then the README RNG
claim **#6**.
